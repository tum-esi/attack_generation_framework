How to build a container testbed network?

  • Create a docker network and connect all containers to this network (this allows containers to discover each other)
  • If persistent storage is in use, make sure to map volumes to folders specified in /persistent/example_app because /persistent has volume mappings to the local computer folder ./persistent
  • (keep in mind)

Containers Page

  • When starting a new container, make sure to specify ports as follows: (1) 8199-8299:8200 for ranges e.g. as with attack-generator, (2) 8300:8300,5900:5900,6080:80 for multiple mappings e.g. as with receiver.
